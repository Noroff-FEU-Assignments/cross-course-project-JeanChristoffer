Web Security

How would you advise a potential client to secure his/her website from web security attacks? Note: the word count for your answer should not be greater than 250 words

I would inform the client about security practices that have been proven to be efficient.

    This are:
  1. Sanitization of data
    Make sure the data your system accepts is in the correct format. This can be achieved in for example NodeJS. Which offers a "sanitize-html package". The sanitize() method is then used to remove any unwanted elements.
  2. Authorization protocol
    Use a well-known and trusted authorization protocol like OneLogin which is hard to crack.
  3. Bounty system
    A bounty system for your security protocols is also a good solution. This involves hiring white hat hackers to test your system’s security and find vulnerabilities in your system.
  4. CI/CD
    CI/CD which stands for Continuous Deployment/Continuous Integration. This process checks every change against security checks. if the changes does not pass the checks gets flagged and prevents the code from going to production.
  5. Network engineer
    Hire a good network engineer. They are tasked with protecting the exchange of packets across the network.
  6. Timeout session
    Implement a timeout session so employees don’t have their credentials or other sensitive data visible in the browser for an extended period of time.
